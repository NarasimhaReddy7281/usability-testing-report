Swiggy Usability Testing Report

Project Overview

This project evaluates the usability of the Swiggy mobile application. The aim is to analyze how users interact with the app, identify usability issues, and suggest improvements to enhance the user experience.

Objective

- Understand user behavior while using Swiggy.
- Identify usability problems.
- Evaluate the efficiency of common tasks.
- Provide recommendations for improving the application.

Testing Methodology

The usability test was conducted by observing users as they completed common tasks in the Swiggy app.

Participants

- 5 participants
- Students and working professionals
- Different levels of experience with Swiggy

Tasks Performed

1. Log in to the application.
2. Search for a restaurant.
3. Add food items to the cart.
4. Apply coupons and discounts.
5. Track an order.

Key Findings

- Users found the search feature useful.
- Coupon selection was sometimes confusing.
- Restaurant menus required excessive scrolling.
- Sponsored listings affected search results.
- Order tracking could be improved.

Recommendations

- Simplify the login process.
- Improve coupon visibility.
- Organize menus with better categories.
- Reduce the prominence of advertisements.
- Provide clearer order updates.


Tools Used

- Google Docs
- Canva
- GitHub
- Swiggy Mobile Application

Conclusion

The project helped identify usability strengths and weaknesses in the Swiggy mobile application. The findings and recommendations can be used to improve the overall user experience.
